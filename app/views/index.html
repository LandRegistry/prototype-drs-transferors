DRS Prototype

Small scale applications (SSA)

Overview

This prototype introduces a new journey for small scale applications, based on how many title numbers a user wants to add. Users are asked to choose between ‘1 to 25’ or ‘26 to 199’. If they select ‘1 to 25’, they continue through the current application journey. If they select ‘26 to 199’, they are taken down a new route designed to make bulk entry and processing more efficient.

In this new journey, users can paste in all their title numbers at once. The service then checks these against internal records and automatically extracts key information, such as property addresses and any restrictions. The results are displayed on a new ‘Manage titles’ page, where users can review and confirm the details.

This page also introduces a change to how part transfers are handled. If a user selects ‘part’ for any title, they are now asked to enter the required additional information directly within the same screen. Previously, this step happened later in the journey on a separate page. Bringing it forward reduces duplication and avoids displaying a large list of titles again later on.

Once users confirm everything is correct, they move to the ‘Disclosable overriding interests’ page. From that point, the rest of the application journey follows the existing service.

  1. Start on the Digital Registration Service start page
  2. Paste in a list of 25 titles to trigger a validation error (you must enter between 26 and 199 titles)
  3. Retry with a new list of 30 valid titles to continue the journey

  1. Start on the Digital Registration Service start page
  2. Paste in a list of 30 titles, including 3 that are unrecognised or unsupported
  3. See an error page showing the failed titles, with instructions to re-enter all title numbers
  4. Retry with a corrected list of 27 valid titles and continue the journey via S1
  5. Download test data (30 titles, 3 errors)
  6. Download test data (corrected list of 27 titles)

Implementation of VCAD Validation Flow (Part 1)

Overview

This prototype demonstrates the new Validating Customer Application Data (VCAD) implementation for DRS. Part 1 focuses on the enhanced submission flow and new application status management system, introducing level 3 validation which compares user-input data against, known data and extracted document data.

Key Changes

  • New "Applications" page (formerly "Saved applications") showing all application statuses
  • Introduction of level 3 validation (VCAD) for document data extraction
  • Enhanced status management: Validating, Rejected, and Accepted statuses
  • Improved application tracking with filtering capabilities
  • New interim page explaining the validation process

Key Change in Routing

Instead of users being routed directly to success/failure pages, all scenarios now direct users through to the new Applications page. This centralised approach allows users to manage and monitor all their applications from a single location. Select application type prototype

This prototype focuses on the initial stage of the application process where users select the type of application they want to make. It aims to guide users to choose the appropriate application type based on their requirements, ensuring that applications are routed to the correct processing path, such as register update, transfer of part, LE, NL or JP1. The prototype explores various designs and flows to achieve this goal effectively.
