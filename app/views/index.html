DRS Prototype

Implementation of VCAD Validation Flow (Part 1)

Overview

This prototype demonstrates the new Validating Customer Application Data (VCAD) implementation for the DRS digital registration service. Part 1 focuses on the enhanced submission flow and new application status management system, introducing level 3 validation which compares user-input data against, known data and extracted document data.

Key Changes

  • New "Applications" page (formerly "Saved Applications") showing all application statuses
  • Introduction of level 3 validation (VCAD) for document data extraction
  • Enhanced status management: Validating, Rejected, and Received statuses
  • Improved application tracking with filtering capabilities
  • New interim page explaining the validation process

Key Change in Routing

Instead of users being routed directly to success/failure pages, all scenarios now direct users through the new Applications page. This centralised approach allows users to manage and monitor all their applications from a single location. Given our assumption that Level 3 (L3) extraction may take an unreasonably long time, which would be impractical for users to wait for validation to complete, this approach offers a more efficient experience. {% from "govuk/components/inset-text/macro.njk" import govukInsetText %} {{ govukInsetText({ text: "We plan to review this routing model once the extraction pilot results are available, to ensure it remains the optimal solution based on real-world data." }) }}

Scenario 1: Successful Submission
  1. User starts on the Review Your Application page
  2. Clicks "Send for Validation" button
  3. Routes to interim page explaining validation process
  4. User clicks "Continue" button
  5. Routes to new Applications page where:
    • Application appears in the top row
    • Status shows as Received
  6. User selects "Open" from action dropdown
  7. Routes to "Application submitted" confirmation page with links to View Applications for detailed information
Scenario 2: Rejected Submission
  1. User starts on the Review Your Application page
  2. Clicks "Send for Validation" button
  3. Routes to interim page explaining validation process
  4. User clicks "Continue" button
  5. Routes to new Applications page where:
    • Application appears in second row
    • Status shows as Rejected
  6. User selects "Open" from action dropdown
  7. Routes to error management system (Part 2 - not included in current prototype)
Scenario 3: Validation In Progress
  1. User starts on the Review Your Application page
  2. Clicks "Send for Validation" button
  3. Routes to interim page explaining validation process
  4. User clicks "Continue" button
  5. Routes to new Applications page where:
    • Application appears in the third row
    • Status shows as
      Validating
  6. User selects "Open" from action dropdown
  7. Routes to "Application Being Validated" page
  8. User clicks "Continue" button
  9. Returns to Applications page with status updating automatically to either "Received" or "Rejected"

Select application type prototype

This prototype focuses on the initial stage of the application process where users select the type of application they want to make. It aims to guide users to choose the appropriate application type based on their requirements, ensuring that applications are routed to the correct processing path, such as register update, transfer of part, LE, NL or JP1. The prototype explores various designs and flows to achieve this goal effectively.
